## Lessons Learned

(Captured in `dev/LEARNINGS.md` as LESSON-019, LESSON-020, LESSON-021.)

- **LESSON-019 Perfection Loop pattern (43 issues closed in one release)** — the ECHO Law 13 utility-first + Law 12 redact() + Law 14 swallowed-error pattern. 5 code-reviewer passes + 3 self-correction rounds to reach ECHO-compliant convergence. Implementation details: `Object.getPrototypeOf` over `v.constructor`; `/api[_]?keys?$/i` for plurals; `Array.isArray` must come before prototype check; naive `grep` over-counts comment lines.
- **LESSON-020 Tauri version alignment to renderer** — workspace Cargo.toml + tauri.conf.json were at 0.2.0 (premature bump from prior drift). Aligned to 0.0.3 at the v0.0.3 release cut per the release-only-versioning discipline.
- **LESSON-021 Dev guard security pattern for `output: "export"`** — with static export, route handlers are compiled to static JSON files at build time. Sensitive data in the response bakes into the Tauri bundle. Fix: gate on `NODE_ENV !== "production"` (dev only); in production, return redacted summary.
- **Two-commit release pattern** — commit 1 for feature work, commit 2 for version bump + CHANGELOG tag. Matches the FID auto-archive pattern from v0.0.2. `scripts/release.py` requires a clean tree.
- **Code-reviewer nits are non-blocking by default** — the v0.0.3 release shipped 4 code-reviewer nits that were addressed in post-release commits. The release tag was already pushed when the nits were raised. Pattern: ship the release, address nits in follow-up commits.
